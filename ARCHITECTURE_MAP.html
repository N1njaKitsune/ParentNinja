Recommended Build Order

Based on priority and dependency chain — the suggested sequence for moving to integration-ready.

Phase A
Foundation — before any integration
  1. Real auth + session management C1, C2
  2. Empty states across all screens C4
  3. Error + loading states C5, C6
  4. Passcode lockout C10
Phase B
Core completeness
  1. Student Profile screen C3
  2. Edit guardian details + change passcode H1, H2
  3. Edit student + re-activate H3, H4
  4. Make-up date picker + duplicate guard C7, C8
  5. Transfer states (pending/approved/rejected) H5
  6. Settings screen H13
Phase C
Live integration
  1. Backend auth wiring
  2. Real-time notifications + preferences H6, H7, H8
  3. Attendance instructor event triggers C9
  4. Stripe live billing H9, H10, H11
Phase D
Polish + launch-ready
  1. Student Profile rich content M3, M4, M5
  2. Shop basket + checkout M11, M12, M13
  3. Biometrics, GDPR, legal M14, P11, P12
  4. Accessibility audit P15
  5. In-app messaging P8
